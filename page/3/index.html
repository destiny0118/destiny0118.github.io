warp-based transformation 变形操作

1. 课题的来源、意义

人工智能生成内容（AIGC，AI Generated Content，AIGC），指的是利用人工智能技术进行内容生成。AI绘画、AI写作等都属于AIGC的具体形式。AIGC可分为音频生成、文本生成、图像生成、视频生成等。在2022年，随着ChatGPT、Stable Disfussion等的出现，拉开了利用人工智能进行内容创作和生产的序幕，随着深度学习模型的不断完善，深度学习理论的不断推进，大模型的出现和商业化，都在助力AIGC的快速发展。动作迁移（Motion Transfer）实现两个人之间的动作迁移，将来自源人物的动作信息迁移到另一个目标人物身上，让其做出相应的动作。动作迁移对于计算机动画领域有着重要意义
动作迁移(Motion Transfer)是计算机图形学和计算机视觉领域的研究方向，它涉及将一个人或对象的动作特征、动作风格或运动样式应用于另一个人或者对象上，以实现动作的自然、真实的传递或者迁移，这个研究方向在很多领域有着广泛的应用和重要的研究意义。动作迁移广泛的应用于电影制作、游戏开发、虚拟现实以及动作制作等领域。电影制作是动作迁移技术的重要应用，在3D电影制作过程中通过使用动作迁移技术将专业动作演员的动作特征应用于虚拟就饿色，以获得更逼真的视觉效果和更流畅的动作表现。在游戏领域，动作迁移可以将真实世界的动作转移到游戏角色身上，获得更真实的游戏体验和更友好的互动效果。在虚拟现实中，动作迁移可以创造更真实的虚拟体验，使用户可以以自然的方式与虚拟环境进行交互。在动画制作过程中，动作迁移可以辅助进行动画创作，使动作制作更加高效。通过将一个角色的动作应用于其他角色身上，可以极大的降低动作制作的时间和成本。
动作迁移涉及多个领域，包括计算机图形学、计算机视觉、深度学习等，研究动作迁移推动了这些领域的发展。同时，这些领域的进步和完善，也为动作迁移带来了更多的可能性。
动作迁移技术因其广泛的应用场景，不仅有着研究意义，在很多领域还有着广泛的现实意义。
动作迁移技术的不断发展和创新将进一步推动这些领域的进步，为生活中的更多场景带来更多新的应用和可能性。首先，动作迁移可以提高视觉效果的真实性和逼真度，通过将专业演员的动作应用于虚拟角色或者对象上，可以使动作看起来更加自然和真是，提高了视觉效果。其次，动作迁移可以节省制作时间和成本，动作迁移技术可以大幅降低制作过程中的成本和时间，不需要针对每一个角色重新创作动作，极大的减少了人力和时间成本。同时，动作迁移技术降低相关技术的门槛，使得每一个人都可以参与内容创作，为内容创作和生产带来了更多的可能性。最后，动作迁移技术可以增强用户体验。在游戏、虚拟现实和增强现实等应用中，动作迁移可以使用户的交互更加直观和自然，增强了用户的沉浸感和体验。

2. 课题的国内外研究概况及发展趋势

  • 准确的姿势估计
  • 真实的图片到图片翻译
    关于动作迁移(motion transfer)这一课题，国内外进行了很多研究工作。早期的方法通常是通过处理现有的视频片段来创作新内容，通过从视频中选择与源人物动作最为接近的帧来进行Video Rewrite。随着深度学习的快速发展，尤其时生成对抗网络的出现及其在图像生成领域所展现的惊人结果，为解决动作迁移这一图像生成问题提供了全新的解决方案和可能性。

自动画人物之间的运动重定向(motion retargte)问题提出以来，涌现了逆运动学(inverse kinematic)求解器和前向运动学等解决方案

EveryBody Dance Now试图源人物和目标人物之间的映射关系来直接进行图片到图片之间的翻译任务，由于两个人物之间身体外形和动作风格的差异。无法获得两个人物进行相同的动作来进行监督学习。由于基于关节点的姿势保留了动作特征，同时去掉了人物身份特征信息，因此选用了关节点姿势作为中间表示。
Villegas et al.研究在骨架之间进行运动重定位，并得到相应骨骼下的3D动画人物。
另一些方法关注于特征分解，MoCoGAN将其分解为内容(在视频中的人物)和动作(人物所表现的动态特征)，通过无监督的对抗训练方式来学习这种分离，并生成目标人物在新动作下的视频。
通过深层的潜在特征来进行图像翻译，

现有的动作迁移方法主要分为图像到图像的翻译(image-to-image translation)，图像翻译将图像中内容从一个图像域转换到另一个图像域，在动作迁移中将姿势标签映射到人物图像，这一方法依赖于生成网络，在生成的图片结果方面容易导致伪影。

还有一些方法基于图像翻译结构设计了更精细的网络模型，用于更通用的以姿势引导的人物图像合成。Pose Guided Person Image Generation以人物参考图像和姿势标签作为条件，以生成更好的细节。
Liquid Warping GAN提出了一个新的策略，通过对人物进行隐式的3D建模，再将3D模型投影到2D，通过一个同一的模型来实现动作迁移，外观迁移和新视角合成的任务。

Synthesizing Images of Humans in Unseen Poses将人体分割为不同部分，并根据目标姿势对其进行变形操作。

视频之间的动作重定向比图像之间的动作重定向更具有挑战性，因为需要学习待转移运动的动态特性和保证时间连续性。人类的感知对于不自然的运动非常敏感，因此合成动作视频比合成高质量的图像更具有挑战性。

在条件图像生成方面，许多工作集中于通过特定的条件，如姿势等引导人物图像的合成。姿势引导的人物图像生成可以产生任意姿势下的人物图像

在动作迁移领域，一般采用无监督的方式进行网络训练，对于源人物的动作，目标人物由于动作风格和身材体型的差异，很难有与源人物动作相匹配的动作图像作为ground truth，来监督网络模型的训练。

3. 课题的研究内容和技术方案

给定两段视频，一段是我们希望合成其外观的目标人物(target person)，另一段是源人物视频(source person)，目标是将其运动应用到目标人物身上。

尽可能的利用已有的信息，原视频人物包含动作语义，对其提取动作骨骼。对动作骨骼进行规范化得到目标人物的动作骨骼，对其身体区域进行规范化得到生成图像的人物动作区域。

基于条件GAN网络的方法，需要得到人体骨骼作为中间表示，在以此来引导最终图像的生成。这种通过编码得到特征的方法，不可避免的会导致信息丢失。图像的数据表示每一个像素点是RGB三通道的值，同时原图像是包含人物的特征信息。设法找到原像素在目标动作下对应的像素位置，将像素值填充到要生成图像的相应位置，便可以得到具有目标动作的人物图像。基于生成网络的方法，生成最终图像的每一个像素点的像素值，这同时带来很大的计算开销和误差。基于光流的动作重定位方法也是利用原像素点信息，但是需要学习一个光流来表示像素的移动方向，这适用于两个动作主体之间的动作差异不大的情况下。如果动作幅度变化差异很大，则光流很难度量这种变化。而基于像素点重定位的方法，则避免了这种情况，不受两者动作差异的影响。同时，不管人物的衣服，还是外观特征，都具有一定的对称性，即使只具有目标人物的正面图片，通过利用相应位置的像素值信息，也可以较好的得到具有指定动作的人物图像。另一方面，可以充分利用注意力机制，只在对应的区域内利用像素点信息。比如，生成图像的人物头部区域利用原图像头部区域的像素点信息。一个弊端是，该方法，无法得到很好的监督。
N表示利用的原像素点个数，通过加权求和来得到目标像素点的值。这一结构可以作为网络的一个分支，利用人物的像素信息，辅助最终人物图像的生成。
$$
target(x_i,y_i)=\sum\limits_{n=1}^{N}source(xi,y_i)
$$

Gitlet支持的主要功能：

  • Saving the contents of entire directories of files. In Gitlet, this is called ==committing==, and the saved contents themselves are called ==commits==.

  • Restoring a version of one or more files or entire commits. In Gitlet, this is called ==checking out== those files or that commit.

  • Viewing the history of your backups. In Gitlet, you view this history in something called the ==log==.

  • Maintaining related sequences of commits, called ==branches==.

  • ==Merging== changes made in one branch into another.

概述

HEAD(the head pointer)跟踪当前分支的最新commit

线性的提交commit是链表数据结构

可以Branch构成Tree结构

类和数据结构

内部结构(Internal Structures)

  • blobs：保存的文件内容，Gitlet会保存文件的不同版本，一个文件可能对应多个blob。
  • trees：目录结构将文件名映射到blob
  • commits：包含log信息，其他元数据（提交日期、作者等），对tree的引用，对父节点的引用

Gitlet在Git基础上进行了简化，将trees嵌入到commits中；只能对两个节点进行merge操作；限制commits包括的数据信息。

  • repository：版本库
  • SHA-1 hash value：通过SHA-1加密哈希函数，根据文件内容计算的哈希值，可以用于文件寻址，或者比较文件内容是否相同
  • working directory
  • staging area

Main

处理命令行输入的命令参数，根据输入的命令调用Repository的相关方法，同时验证命令行参数输入的有效性。如果输入不正确的命令，或者命令对应的参数不正确，则打印相关错误信息。

Method

  1. public static void main(String[] args)：程序入口，根据输入的命令参数执行相应的过程
  2. public static void validateNumArgs(String cmd, String[] args, int n)：验证参数个数是否正确，对于checkout命令单独处理，因为该命令可以多种不同参数个数的调用格式

Commit

处理每次commit任务，包括创建commit数据结构信息(包括提交信息，提交日期，父节点，跟踪的文件名)

Fields

  1. private String message
  2. private String date
  3. private LinkedList<String> parentID
  4. private Map<String, String> dict

Method

  1. public Commit()：默认构造方法，再init时候生成初始commit信息
  2. public static String getCommitHashVal(), getCommitHashVal(String branch)：无参数时默认读取当前HEAD指向的commit，否则读取相应分支下的最近一次commit
  3. public static Commit readCommit(String commitHashVal)：根据commit的哈希值读取对应的文件，反序列化得到Commit数据结构
  4. public void setInfo(String message, LinkedList<String> parentID)：设置Commit的相关信息，在Commit时，首先读取父节点的数据结构，对父节点的数据结构进行处理得到当前commit
  5. public void writeCommit()：将Commit数据结构利用序列化存储到一个暂时的文件当中，计算文件的hash，然后再将文件根据hash移动到指定位置。更新CommitTrie和CommitGraph数据结构，将hash值写到当前分支中，以跟踪当前分支的最新一次提交。

CommitGraph

Fields

private Map<String, LinkedList<String>> g：邻接链表存储，存储当前commit的父节点信息

Mehod

  1. public static void DFS(String cur)：深度优先搜索，从当前cur表示的commit出发，不断递归其第一个父节点，直至到达初始提交
  2. public static void BFS(boolean findMessage, String message)：参数作为标志，如果findMessage为true，对应find命令，找到是否有提交信息为message的commit；否则对应global-log命令，打印处所有的commit信息。
  3. private static void printCommit(String cur, Commit commit)：格式打印当前cur表示的commit的相关信息
  4. public static String latestCommonAncestor(String p, String q)：找到p和q对应的commit的最近公共父节点。
  5. public void addEdge(String u, String v)：添加u到v的边，其中u为当前commit，v为父节点

CommitTrie

保存commit的hash值的数据结构，字典树实现，用于在给定uid时快速查找到对应的commit

Fields

  • LinkedList<String> commit：存在于当前节点的commit的hash值
  • CommitTrie[] son：子节点

Method

  1. public static void insert(String commitID, CommitTrie root)：将commit对应的hash插入到字典树当中
  2. public static String query(String commitID, CommitTrie root)：查询具有给定uid的commit

Repository

remoteRepo

处理add-remote、rm-remote、push、fetch、pull命令，完成本地仓库与远端仓库的交互逻辑，远端仓库在本地模拟，给定路径作为远端仓库地址。

Fields

  • private Map<String, String> remoteDir：保存(远端仓库名，远端地址)，这一数据结构保存在CONFIG中

Method

  1. public static void addRemote(String name, String path)：添加远端仓库，并在本地.gitlet/refs/remotes/目录下添加远端仓库名文件夹，用以存储远端仓库的分支
  2. public static void rmRemote(String name)：从remoteDir数据结构中移除远端仓库，并将本地.gitlet/refs/remotes/目录下的用以跟踪远端仓库的文件夹及其内的分支删除
  3. public static void push(String name, String branch)：将本地当前分支的修改推送到远端仓库，远端分支的头节点需为当前分支的历史节点。将远端分支和本地分支之间提交的commits推送到远端仓库，远端分支跟踪的头节点需进行更新。
  4. private static void copyCommit(String commitHash, File src, File dst)：将commitHash跟踪的所有文件和存储commit数据结构的文件从src目录复制到dst目录
  5. public static void fetch(String name, String branch)：将远端仓库的修改拉到本地仓库，根据远端仓库refs/heads/branch记录的commit与本地仓库refs/remote/name/branch下记录的commit，可以追踪本地没有的commit。
  6. public static void pull(String name, String branch)：与fetch相比，该命令还需将拉取的分支与本地分支合并(代码复用，利用Repository.merge实现)。

ST表

离散表（Sparse Table，ST），主要用来解决最大值/最小值查询(Range Minimum/Maximum Query，RMQ)，可以快速的查询区间内的最大值最小值。lookup[i][j]表示从下标i开始长度为$2^j$的区间的最小值。

区间[i,j]长度为L=j-i+1，得到k使得$2^k<=L并且2^{k+1}>L$，考虑两个子区间$[i,i+2^k-1]$和$[j-2^k+1,j]$，两个区间的长度为$2^k$，这两个区间重叠。则$min(i,j)=min(lookup[i][k],lookup[j-2^k+1][k])$。

计算lookup，$lookup[i][j]=min(lookup[i][j-1],lookup[i+2^{j-1}][j-1])$，将长度为$2^j$的区间分为长度为$2^{j-1}$的两个区间。
$$
计算lookup需要O(nlogn)的时间和空间复杂度\
查询O(1)复杂度
$$

适用于数组不变、多次查询

其实ST表不仅能处理最大值/最小值，凡是符合结合律且可重复贡献的信息查询都可以使用ST表高效进行。什么叫可重复贡献呢？设有一个二元运算$f(x,y)$，满足 $f(a,a)=a$，则$f$是可重复贡献的。显然最大值、最小值、最大公因数、最小公倍数、按位或、按位与都符合这个条件。可重复贡献的意义在于，可以对两个交集不为空的区间进行信息合并。

2763. 所有子数组中不平衡数字之和

方法一：枚举

题目规定sarr[i+1] - sarr[i] > 1为不平衡数字。

枚举子数组，i作为左侧边界，同时不断扩展右侧边界。在扩展右侧边界时，记录不平衡数字变化情况。如果x-1和x+1都存在，添加x使得不平衡数字减1。都不存在时，将使得当前数字与某一个数字构成不平衡数字。

方法二：贡献法

统计每个数字对不平衡数字的贡献数，x=nums[i]，子数组中如果存在x-1，则x肯定无法成为不平衡数字。同时为了避免重复统计，子数组中可以存在x+1。这考虑的是x作为不平衡数字的较大数字的贡献度。子数组左边可以包含x。同时，每个数字可能作为子数组的最小值，这种情况排序后的子数组分别有n,n-1,..,1个。

2281. 巫师的总力量和

求的是子数组的和*最小值，然后所有子数组的情况再进行求和

$$
\begin{aligned}
&sum是strength的前缀和，ss是sum的前缀和\
&对于[L+1,R-1]范围内，和为sum[R]-sum[L-1]\
&[L+1,R-1]范围内的所有子数组：\ \ \
&\sum_{p=L+1}^{i}\sum_{q=i}^{R-1}(sum[q+1]-sum[p])\
&=(i-L)\sum_{q=i}^{R-1}sum[q+1]-(R-i)\sum_{p=L+1}^{i}sum[p]\
&=(i-L)[sum[i+1]…sum[R]]-(R-i)[sum[L+1]…sum[i]]\
&=(i-L)(ss[R+1]-ss[i+1])-(R-i)(ss[i+1]-ss[L+1])
\end{aligned}
$$
